# 🐝 Swarm Project Journal

## July 2, 2026 — The Big One

Started the week chasing LLM-JEPA. Finished it with a fully modular multi-agent swarm that can reason about what tools it needs and execute in pipeline mode. Here's how it went down.

### June 30 — The LLM-JEPA Era

Ran Runs 1-4 of the LLM-JEPA experiment. Variance weight 1.0 still wasn't enough — the predictor spread kept decoupling from the encoder collapse. Time to pivot.

### July 1 — Birth of the Swarm

Scrapped LLM-JEPA entirely. Replaced it with a multi-agent swarm concept. Made the search engine agnostic (DDG, SearXNG, Google), added JSON config, custom prompts. Chaos monkey caught a few bugs early — empty goal guard, JSON routing fix, malformed config handling. Added the ASCII architecture diagram to README because every project needs a good diagram.

Auto-save to markdown files was the first real quality-of-life win. No more lost outputs.

### July 2 — The Marathon

This was the day. **27 commits in ~12 hours.**

**Morning/Afternoon:**
- Scratchpad (write-only SQLite) — workers log findings, never read each other's context. This was the key insight.
- First benchmark runs
- Refactored the monolithic `swarm2.py` into the modular `swarm/` package
- Replaced regex complexity estimation with a model-based approach (DeepSeek V4 Flash)
- Extracted CLI into runner + output modules

**Evening:**
- Post-commit hook for auto-testing — every commit runs chaos monkey + benchmark in background
- Orchestrator synthesis — the "boss reads the room" step that ties all worker outputs together
- Modular tool system — `swarm/tools/` with base classes, registry, per-tool modules
- Vision tool (Gemma4 reads images), Python exec tool, File reader tool
- Tool bundles — preflight assigns `vision`/`code`/`files`/`default` based on question analysis

**Late Night:**
- LLM-based bundle assignment — the orchestrator model (DeepSeek) reasons about what tools each worker needs instead of hardcoded rules
- Pipeline mode — workers can have sequential dependencies (vision reads image → code computes)
- Preserved the old research version as `demo-swarm/`
- Overhauled README and AGENTS.md to reflect the actual architecture
- GAIA benchmark eval harness built and tested (2/3 on image-based questions)
- MuJoCo integration working on the side
- Jotted down the TUI idea for later
